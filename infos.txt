DIO LOTO 

1. Project Overview  
DIO LOTO is an online lottery gaming platform where users can bet money for a chance to win rewards.  

The application is already developed and allows users to:  
- Register and log in.  
- Access a dashboard where they can check their balance and winnings.  
- Top up their account via a **reseller**.  
- Place bets to participate in lottery games.  
- Instantly receive their winnings in case of a victory.  

The system is based on three categories of actors:  

1.1 System Actors  

🔹 User  
- Email  
- Last name  
- First name  
- Password  
- Balance (available money for playing)  
- Winnings (money won from bets)  

🔹 Administrator (Admin)  
- Last name  
- First name  
- Email  
- Password  
- Balance  

🔹 Reseller  
- Last name  
- First name  
- Email  
- Password  
- Reseller balance (money available for user top-ups)  
- Earnings balance (money earned from activity)  

2. System Operation  

2.1 User Account Top-Up  
To play, the user must have a balance. They can top up their account by contacting a reseller and sending money via Mobile Money along with their email.  

1. The user contacts a reseller.  
2. They send a Mobile Money payment, specifying their email.  
3. The reseller confirms the payment and credits the user's account.  
4. The user sees their updated balance on their dashboard.  

2.2 Fund Management  
- Resellers need funds to be able to top up user accounts.  
- Only administrators can top up reseller balances.  
- Administrators oversee all transactions within the system.  

3. Development of an Administration Interface  

I want to create a web administration application to efficiently manage the system.  

3.1 Administrator Authentication  
- Before accessing the dashboard, an admin must authenticate.  
- Once logged in, they can access the Admin Dashboard.  

3.2 Dashboard Functions  

🔹 User Management
- Display a list of all users.  
- View each user’s information (balance, winnings, bets, etc.).  

🔹 Reseller Management  
- List all resellers.  
- View each reseller’s balance.  
- Top up a reseller's account.  
- Block/unblock a reseller (make them inactive/active in the system).  
- Modify or delete a reseller.  

🔹 Transaction Monitoring
- Display all transactions:  
  - User top-ups via resellers.  
  - User bets.  
  - Admin top-ups to resellers.  
- Generate a daily summary:  
  - Total top-ups for the day.  
  - Total withdrawals.  
  - Remaining balance for each reseller.  
  - Total amount bet by users.  

🔹 Notification System 
- Display new registrations (users and resellers).  
- Show recent transactions.  

4. Development Objective  

🎯 Create a complete administration system for DIO LOTO, including:  
✅ A secure interface for administrators.  
✅ Efficient management of users and resellers.  
✅ Accurate transaction tracking.  
✅ Full control over financial operations.




1. Présentation du Projet
DIO LOTO est une plateforme de jeu de loto en ligne où les utilisateurs (users) peuvent miser de l'argent pour tenter de gagner des gains.  

L’application est déjà développée et permet aux utilisateurs de :  
- S'inscrire et se connecter.  
- Accéder à un tableau de bord où ils peuvent consulter leur solde et leurs gains.  
- Recharger leur compte via un **revendeur**.  
- Miser leur argent pour participer aux jeux de loto.  
- Recevoir instantanément leurs gains en cas de victoire.  

Le système repose sur trois catégories d’acteurs:  

1.1 Les acteurs du système

🔹 Utilisateur (User)
- Email  
- Nom  
- Prénom  
- Mot de passe  
- Solde (argent disponible pour jouer)  
- Gain (argent gagné grâce aux mises)  

🔹 Administrateur (Admin)
- Nom  
- Prénom  
- Email  
- Mot de passe  
- Solde  

🔹 Revendeur (Reseller)
- Nom  
- Prénom  
- Email  
- Mot de passe  
- Solde revendeur (argent qu'il détient pour recharger les utilisateurs)  
- Solde gain (argent gagné grâce à son activité)  

2. Fonctionnement du Système

2.1 Rechargement du compte utilisateur
Pour jouer, l’utilisateur doit avoir du solde. Il peut recharger son compte en contactant un revendeur et en lui envoyant de l’argent via Mobile Money accompagné de son email.  

1. L’utilisateur contacte un revendeur.  
2. Il lui envoie un paiement Mobile Money en indiquant son email.  
3. Le revendeur valide la réception et crédite le compte de l’utilisateur.  
4. L’utilisateur voit son solde mis à jour sur son tableau de bord.  

2.2 Gestion des fonds
- Les revendeurs ont besoin de fonds pour pouvoir recharger les utilisateurs.  
- Seuls les administrateurs peuvent recharger le solde des revendeurs.  
- Les administrateurs supervisent toutes les transactions du système.  

3. Développement d’une Interface d’Administration 

Je souhaite créer une application web d’administration pour gérer efficacement le système.  

3.1 Authentification des administrateurs
- Avant d’accéder au tableau de bord, un admin doit s’authentifier.  
- Une fois connecté, il accède au Dashboard Administratif.  

3.2 Fonctions du Dashboard 

🔹 Gestion des Utilisateurs  
- Afficher la liste de tous les utilisateurs.  
- Consulter les informations de chaque utilisateur (solde, gains, mises, etc.).  

🔹 Gestion des Revendeurs  
- Lister tous les revendeurs.  
- Voir le solde de chaque revendeur.  
- Recharger le compte d’un revendeur.  
- Bloquer/débloquer un revendeur (rendre inactif/actif dans le système).  
- Modifier ou supprimer un revendeur.  

🔹 Suivi des Transactions  
- Afficher toutes les transactions effectuées :  
  - Recharges des utilisateurs via les revendeurs.  
  - Mises des utilisateurs.  
  - Recharges des admins vers les revendeurs.  
- Générer un récapitulatif journalier :  
  - Total des recharges du jour.  
  - Total des retraits.  
  - Montant restant pour chaque revendeur.  
  - Total des sommes mises en jeu par les utilisateurs.  

🔹 Système de Notifications
- Affichage des nouvelles inscriptions (utilisateurs et revendeurs).  
- Affichage des transactions récentes.
